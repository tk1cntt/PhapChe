## Warning 01: Schema Mismatch

vault-service.ts references `size` and `contentType` in Prisma select/create queries (lines 60-61, 92-93, 185-186) but VaultFile model in schema.prisma lacks these fields. This causes runtime Prisma query failures.

## Warning 02: Missing Variable Schema Builder

Template creation form (admin/templates/new/page.tsx) only has fields for matterTypeKey, label, description, content. The server action (actions.ts line 43) hardcodes `variableSchema: []`. Admin cannot define template variables through the UI.

## Variable Schema Structure

```typescript
type TemplateVariable = {
  key: string;       // e.g. "company_name"
  label: string;     // e.g. "Tên công ty"
  required: boolean;
  type: 'text' | 'number' | 'date' | 'select';
};
```

The form should allow adding/removing rows with these fields.
</context>
