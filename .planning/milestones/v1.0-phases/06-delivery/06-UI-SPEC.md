## Page Structure

### Customer Request Detail Page (`/customer/requests/[requestId]`)

Follows specialist request detail page layout pattern.

```
PageHeader: "Yêu cầu pháp lý" + request title
Card: Request metadata (status badge, customer name, matter type, created date)
Card: Final Documents list — ONLY documents where request.status = 'approved'
  - Each row: document name, template label, version, created date, [Tải xuống] button
  - No drafts, no internal notes, no reviewer comments visible
Card: Close Request section (specialist/coordinator only) — "Đóng hồ sơ" button + reason dialog
```

### Visual Differences from Specialist View

| Element | Specialist View | Customer View |
|---------|-----------------|---------------|
| Documents section | All versions (draft, review, approved) | Only final (approved) documents |
| Vault files | Shows customer uploads + internal files | Hidden completely |
| Internal notes | Visible in document content | Hidden |
| Reviewer comments | Visible in review flow | Hidden |
| Generate draft | Present for in-progress status | Absent |
| Close request | Present in delivered status | Absent (specialist/coordinator only) |
