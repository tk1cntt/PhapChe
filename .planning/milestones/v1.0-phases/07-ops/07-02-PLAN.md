<threat_model>
## Trust Boundaries
| Boundary | Description |
|---|---|
| Browser query string -> server page | User-controlled filters enter the server-rendered page. |
| Server page -> ops service | Page requests privileged operational DTOs from the service boundary. |
| Ops DTO -> rendered HTML | Sensitive backend operational data is displayed to admins. |

## STRIDE Threat Register
| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-07-02-01 | Information Disclosure | `/admin/ops/page.tsx` | mitigate | Always call `requireAppSession`, `parseOpsFilters`, and `getOpsDashboard`; never query Prisma directly in the UI page. |
| T-07-02-02 | Tampering | filter form/query params | mitigate | Submit filters through URL params but trust only service-normalized filter DTO; render safe error state for invalid filters. |
| T-07-02-03 | Integrity | dashboard/list metrics | mitigate | Render counts/list/workload/SLA only from one `getOpsDashboard` payload; no hard-coded fixtures or client recalculation. |
| T-07-02-04 | Information Disclosure | request table cells | mitigate | Render identifiers, labels, statuses, assignees, and ages only; no raw legal content, reviewer comments, document text, vault keys, or file contents. |
</threat_model>
