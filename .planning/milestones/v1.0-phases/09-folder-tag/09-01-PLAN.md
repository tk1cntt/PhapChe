Add to prisma/schema.prisma (place new models after VaultFile for grouping):

1. Folder model:
```prisma
model Folder {
  id              String           @id @default(cuid())
  workspaceId     String
  name            String
  parentId        String?
  createdAt       DateTime         @default(now())
  updatedAt       DateTime         @updatedAt
  workspace       Workspace        @relation(fields: [workspaceId], references: [id])
  parent          Folder?          @relation("FolderHierarchy", fields: [parentId], references: [id])
  children        Folder[]         @relation("FolderHierarchy")
  vaultFileFolders VaultFileFolder[]

  @@index([workspaceId])
  @@index([parentId])
  @@unique([workspaceId, parentId, name])
}
```

2. Tag model:
```prisma
model Tag {
  id           String         @id @default(cuid())
  workspaceId  String
  key          String
  label        String
  createdAt    DateTime       @default(now())
  workspace    Workspace      @relation(fields: [workspaceId], references: [id])
  vaultFileTags VaultFileTag[]

  @@unique([workspaceId, key])
  @@index([workspaceId])
}
```

3. Join tables:
```prisma
model VaultFileFolder {
  vaultFileId  String
  folderId     String
  createdAt    DateTime   @default(now())
  vaultFile    VaultFile  @relation(fields: [vaultFileId], references: [id])
  folder       Folder     @relation(fields: [folderId], references: [id])

  @@id([vaultFileId, folderId])
  @@index([folderId])
}

model VaultFileTag {
  vaultFileId  String
  tagId        String
  createdAt    DateTime   @default(now())
  vaultFile    VaultFile  @relation(fields: [vaultFileId], references: [id])
  tag          Tag        @relation(fields: [tagId], references: [id])

  @@id([vaultFileId, tagId])
  @@index([tagId])
}
```

4. Add back-relations to Workspace:
- `folders Folder[]`
- `tags Tag[]`
- `vaultFileFolders VaultFileFolder[]`
- `vaultFileTags VaultFileTag[]`

5. Add back-relations to VaultFile:
- `vaultFileFolders VaultFileFolder[]`
- `vaultFileTags VaultFileTag[]`

Do NOT modify any existing model fields. Do NOT change enum values. Do NOT add new enums.
</action>

### Task 3: Classification service

<read_first>
- src/lib/documents/template-service.ts (analog: workspace-scoped CRUD with audit)
- src/lib/documents/vault-service.ts (analog: $transaction with audit + RBAC)
- .planning/phases/09-folder-tag/09-PATTERNS.md (lines 92-155 — exact imports + patterns)
- .planning/phases/09-folder-tag/09-CONTEXT.md (D-05..D-07 service decisions)
</read_first>

<action>
Create src/lib/documents/classification-service.ts with the following exports:

1. `createFolder(session, input: { workspaceId, name, parentId? })` — RBAC: admin only. Creates Folder. Emits `recordAuditEvent` with action `folder.created`, targetType `VAULT_FILE`, targetId=folder.id, metadataSummary `name=<sanitized>; parentId=<or root>`. Throws `'FOLDER_DUPLICATE'` if name+parentId combo exists.

2. `listFolders(session, workspaceId, parentId?)` — RBAC: workspace access. Returns Folder[] with `_count` for children + vaultFileFolders. Ordered by name asc.

3. `moveFileToFolder(session, input: { vaultFileId, folderId })` — RBAC: admin only. Inside $transaction: upsert VaultFileFolder (idempotent), emit `recordAuditEvent` with action `vault_file.moved_to_folder`, targetType `VAULT_FILE`, targetId=vaultFileId, metadataSummary `folderId=<id>; folderName=<name>`. Throws `'VAULT_FILE_NOT_FOUND'`, `'FOLDER_NOT_FOUND'`, `'WORKSPACE_MISMATCH'`.

4. `createTag(session, input: { workspaceId, key, label })` — RBAC: admin only. Throws `'TAG_DUPLICATE'` if (workspaceId, key) exists. Audit `tag.created`.

5. `listTags(session, workspaceId)` — RBAC: workspace access. Returns Tag[] with `_count.vaultFileTags`. Ordered by key asc.

6. `tagFile(session, input: { vaultFileId, tagId })` — RBAC: admin only. $transaction with audit. Idempotent upsert. Audit `vault_file.tagged`.

7. `untagFile(session, input: { vaultFileId, tagId })` — RBAC: admin only. $transaction. Audit `vault_file.untagged`. Throws `'VAULT_FILE_TAG_NOT_FOUND'` if missing.

8. `listFileClassifications(session, workspaceId)` — RBAC: workspace access. Returns `Array<{ vaultFile: VaultFile, folders: Folder[], tags: Tag[] }>`.

Top of file: copy `isAdmin(session)` helper from template-service.ts lines 7-9. Throw `'FORBIDDEN'` for non-admin mutations. Throw `'FORBIDDEN'` for reads when `canAccessWorkspace` returns false.

All audit metadataSummary strings must be ≤500 chars and contain NO file content, NO storageKey, NO PII.
</action>
