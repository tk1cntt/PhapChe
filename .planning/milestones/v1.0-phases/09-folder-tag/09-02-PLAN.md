### Task 3: Server actions

<read_first>
- src/app/admin/templates/[templateId]/actions.ts (analog: server actions with revalidatePath)
- src/app/admin/templates/new/actions.ts (analog: useFormState-compatible `{ errors, message }` shape)
- src/lib/documents/classification-service.ts (from plan 09-01 — the 8 service functions)
</read_first>

<action>
Create src/app/admin/vault/actions.ts with `'use server'` directive. Exports:

1. `createFolderAction(prevState, formData)` — useFormState-compatible. Returns `{ errors?: { name?, parentId? }, message? }`. Extracts `name`, `parentId` from FormData. Validates `name.length > 0` and `name.length <= 80`. Calls `createFolder(session, { workspaceId, name, parentId: parentId || undefined })`. On success: revalidatePath('/admin/vault'). On error: return `{ message: err.message }`.

2. `createTagAction(prevState, formData)` — same pattern. Validates `key` (regex `/^[a-z0-9_-]{1,32}$/`) and `label` (1-80 chars). Calls `createTag`. Re-validates.

3. `moveFileToFolderAction(formData)` — simple `{ error?, success? }` shape. Extracts `vaultFileId`, `folderId`. Calls `moveFileToFolder`. Re-validates.

4. `tagFileAction(formData)` — same as moveFileToFolderAction but for tagFile.

5. `untagFileAction(formData)` — same shape. Calls `untagFile`.

All actions start with `const session = await requireAppSession();` and check `session.roles.includes('coordinator_admin') || session.roles.includes('super_admin')` — if not, return `{ error: 'FORBIDDEN' }` (or `{ message: 'FORBIDDEN' }` for useFormState actions).
</action>
