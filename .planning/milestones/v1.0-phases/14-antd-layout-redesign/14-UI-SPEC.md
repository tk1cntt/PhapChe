## Spacing Scale

Existing project uses Tailwind spacing (multiples of 4). Ant Design has its own spacing tokens. This contract aligns both.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding |
| sm | 8px | Compact element spacing, Menu item gap |
| md | 16px | Default element spacing, Card padding |
| lg | 24px | Section padding, layout gaps |
| xl | 32px | Major section breaks, PageHeader padding |
| 2xl | 48px | Page-level sections |
| 3xl | 64px | Extended page margins |

Ant Design token mapping (set via `ConfigProvider theme.token`):

| Ant Design Token | Value | Notes |
|------------------|-------|-------|
| `paddingXXS` | 4px | xs |
| `paddingXS` | 8px | sm |
| `paddingSM` | 12px | antd default; leave as-is |
| `padding` | 16px | md |
| `paddingMD` | 20px | antd default; leave as-is |
| `paddingLG` | 24px | lg |
| `marginXXS` | 4px | xs |
| `marginXS` | 8px | sm |
| `marginSM` | 12px | antd default |
| `margin` | 16px | md |
| `marginMD` | 20px | antd default |
| `marginLG` | 24px | lg |
| `marginXL` | 32px | xl |
| `marginXXL` | 48px | 2xl |

Exceptions: None. Ant Design uses a different internal scale; the contract above ensures Tailwind utility classes (used for layout spacing per D-06) match the 8-point system already in use.

**Source:** Existing code (`globals.css`, `ui.tsx`, `admin-shell.tsx`) uses spacing values 4, 8, 16, 24, 32, 48, 64.

## Typography

| Role | Size | Weight | Line Height | Ant Design Token |
|------|------|--------|-------------|------------------|
| Body | 14px | 400 (regular) | 1.5 | `fontSize: 14` |
| Label / Small / Table header | 13px | 600 (semibold) | 1.3 | Not set (use component-level) |
| Heading (page title) | 30px | 600 (semibold) | 1.15 | Not set (use component-level) |
| Display / Brand header | 22px | 600 (semibold) | 1.2 | Not set (use component-level) |

Typography rules:

- **Two weights only:** 400 (regular) for body text, 600 (semibold) for headings, labels, nav items, badges, table headers.
- **Line height 1.5** for body text (existing `globals.css` `line-height: 1.5`).
- **Line height 1.2** for display/heading text.
- Ant Design `fontSize` seed token set to 14px for consistency.
- Page titles (h1) at 30px semibold remain as-is from existing `PageHeader` component pattern.
- Badge/label text at 13px semibold remains as-is from existing pattern.
- Table header at 13px semibold (consolidated from previous 12px). Use uppercase with 0.08em tracking as-is from existing pattern.

**Source:** Extracted from `ui.tsx` (Button 14px, Badge 13px, Table header 12px originally, PageHeader title 30px), `admin-shell.tsx` (brand header 22px), `globals.css` (body line-height 1.5). Table header consolidated from 12px to 13px to fit within the 4-size limit.

## Color

| Role | Value | Usage | Ant Design Token |
|------|-------|-------|------------------|
| Dominant (60%) | `#F8FAFC` | Page background (Layout, body) | `colorBgLayout: '#F8FAFC'` |
| Secondary (30%) | `#FFFFFF` | Cards, sidebar (Sider), nav (Menu), content containers | `colorBgContainer: '#FFFFFF'` |
| Accent (10%) | `#0F766E` | Reserved for specific accent elements (see list below) | `colorPrimary: '#0F766E'` |
| Destructive | `#DC2626` | Destructive actions only (delete, revoke, reject) | `colorError: '#DC2626'` |
| Text primary | `#0F172A` | Headings, body text, Menu item labels | `colorText: '#0F172A'` |
| Text secondary | `#475569` | Descriptions, secondary labels, placeholder | `colorTextSecondary: '#475569'` |
| Border | `#E2E8F0` | Card borders, table borders, divider lines, Sider border | `colorBorder: '#E2E8F0'` |
| Muted background | `#F1F5F9` | Hover states, disabled backgrounds, table header | `colorFillTertiary: '#F1F5F9'` |
| White | `#FFFFFF` | Text on accent/primary backgrounds | `colorWhite: '#FFFFFF'` |

**Accent (`#0F766E`) reserved for:**
- Primary buttons (Button type="primary")
- Active navigation item (Menu selected item background/text)
- Links / anchor text
- Page header brand text ("GitNexus Legal")
- Focus ring / outline
- Badge semantic tone "accent"
- Selection highlight color (`::selection`)
- Table sort indicator

Accent is NEVER used for: body text, regular card borders, secondary buttons, disabled elements, page backgrounds, destructive actions.

**Source:** CONTEXT.md D-05, existing `globals.css` variables, existing `ui.tsx` and `admin-shell.tsx` hex values.

## Visual Focal Point

The primary visual focal point on every screen is the page content area (Layout.Content containing `{children}`). Sidebar navigation and header breadcrumbs are supporting chrome. All content actions (forms, tables, buttons) are placed within the content area; the sidebar and header serve only as navigation context and spatial orientation.

## Interaction Contracts

### Sidebar Responsive Behavior
- Desktop (>= lg, 992px): Sider expanded, shows icons + labels
- Tablet (md, 768px-991px): Sider collapses to icon-only (collapsedWidth=64)
- Mobile (< 768px): Sider hidden, trigger button visible to toggle drawer-style overlay
- Implementation: `Sider breakpoint="lg" collapsedWidth={64}` handles desktop/tablet. For mobile, use antd's built-in responsive behavior or wrap in a custom hook.

### Breadcrumb Generation
- On every route group layout mount, derive breadcrumb items from `pathname`:
  1. Split `pathname` by `/`
  2. Filter out empty segments and route group segments (parenthesized, e.g., `(admin)`)
  3. Map each segment via centralized config to Vietnamese label
  4. Build cumulative href for each breadcrumb item
  5. Set last item as `disabled` (current page, not clickable)

### Navigation Active State
- `Menu.selectedKeys` is derived from `usePathname()` to match the current route.
- Use pathname prefix matching: if pathname starts with `/admin/users`, highlight "Nguoi dung" in menu.
- Sider automatically highlights the correct item via `selectedKeys`.

### Page Transition
- No page transition animations in this phase. Ant Design Layout renders content immediately.
- Future consideration: `Layout.Content` with a simple CSS fade-in transition (not in scope).

### Global Feedback (message/notification)
- `App` component provides `message`, `modal`, `notification` via hooks (`useMessage`, `useModal`, `useNotification`).
- Existing page success/error toasts should migrate from current pattern to antd `message.success()` / `message.error()`.
- Place a single `{contextHolder}` from `useMessage()` in the root layout or each layout.
