## Phase UI Scope

Phase 1 UI is foundation/admin only:

- Admin user management for roles: `customer`, `specialist`, `reviewer`, `coordinator_admin`, `super_admin`.
- Workspace/organization list and membership visibility for SME isolation.
- Request shell list/detail with backend-owned status display.
- Workflow transition controls only as server-backed actions, never raw status editing.
- Audit event list/timeline for critical Phase 1 actions.
- Permission-denied and empty states for role/workspace boundaries.

Out of scope:

- Intake chat/form UI.
- Reviewer split view or checklist UI.
- Legal Vault file browser, uploads, previews, downloads.
- Delivery UI.
- Ops dashboard/SLA/workload analytics.
- OCR, e-sign, AI, billing, workflow builder.

Source: CONTEXT.md D-01..D-17, deferred ideas; ROADMAP.md Phase 1 boundary.

## Layout and Interaction Contract

| Area | Contract |
|------|----------|
| App shell | Single admin shell with top header and left navigation on desktop. Collapse to top navigation on small screens. |
| Page width | Use full width with `max-width: 1280px`; 24px page padding desktop, 16px mobile. |
| Navigation | Primary items: Users, Workspaces, Requests, Audit. Hide items by role for UX only; server still enforces access. |
| Tables | Use dense admin tables with sticky header when useful. Columns must not expose sensitive legal content. |
| Forms | Use single-column forms for create/edit user, role, workspace membership. Labels always visible. |
| Status changes | Present allowed transition buttons from backend result only. Do not render arbitrary status dropdown. Require reason textarea for assignment changes, cancellation, and revision-like transitions when available. |
| Audit visibility | Audit row shows timestamp, actor, workspace, action, target, correlation id, metadata summary. Never show full legal content. |
| Loading state | Use neutral skeleton rows or `Đang tải dữ liệu...`; do not use blocking full-page spinner for table refresh. |
| Disabled state | Disabled controls must include helper text explaining missing permission or unavailable transition. |
| Permission-denied state | Show concise Vietnamese message and safe next step. Do not reveal whether hidden resource exists. |
| Responsive minimum | All forms usable at 360px width. Tables may horizontally scroll with first identifier column visible. |
| Accessibility | Keyboard focus visible on all interactive elements. Dialogs trap focus. Touch targets minimum 44px for icon-only controls. |

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline helper gaps |
| sm | 8px | Compact element spacing, table cell vertical padding |
| md | 16px | Default element spacing, form field gaps, card padding mobile |
| lg | 24px | Section padding, page padding desktop, card padding desktop |
| xl | 32px | Layout gaps between major groups |
| 2xl | 48px | Major section breaks, empty state vertical spacing |
| 3xl | 64px | Page-level spacing, not used inside dense admin tables |

Exceptions: icon-only touch targets must be 44px minimum; focus ring offset may be 2px; 1px borders allowed.

Source: default 8-point scale, adapted for admin UI density.

## Typography

Use exactly four sizes and two weights.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Label | 14px | 600 | 1.4 |
| Body | 16px | 400 | 1.5 |
| Heading | 20px | 600 | 1.2 |
| Display | 28px | 600 | 1.2 |

Rules:

- Use 14px for table metadata, badges, labels, helper text.
- Use 16px for body text, form input text, table primary cell text.
- Use 20px for page section headings and card titles.
- Use 28px only for page title. No larger marketing display text in Phase 1.
- Use only 400 and 600. No 500, 700, 800.

Source: default admin typography; no existing tokens found.

## Color

Use shadcn-compatible neutral base values in Tailwind tokens or CSS variables.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #F8FAFC | App background, page canvas, empty safe space |
| Secondary (30%) | #FFFFFF | Cards, tables, dialogs, form surfaces, sidebar/nav surfaces |
| Accent (10%) | #0F766E | Primary CTA, active nav item, focus ring, selected filter, approved/success status emphasis |
| Destructive | #DC2626 | Destructive actions only: deactivate user, remove membership, cancel request |

Accent reserved for: primary CTA buttons, active navigation indicator, visible keyboard focus ring, selected filter chips/tabs, success/approved status badge emphasis. Accent must not color every link, every icon, or decorative borders.

Additional semantic tokens:

| Token | Value | Usage |
|-------|-------|-------|
| Text primary | #0F172A | Main text |
| Text secondary | #475569 | Metadata, helper text |
| Border | #E2E8F0 | Card/table/input borders |
| Muted surface | #F1F5F9 | Table header, disabled backgrounds, subtle badges |
| Warning | #D97706 | Workflow states needing attention: triage, pending_review, revision_required |
| Info | #2563EB | Informational status only; not primary CTA |

Source: manual Tailwind/shadcn-compatible defaults; legal/security admin product needs calm neutral palette.

## Component Inventory

| Component | Contract |
|-----------|----------|
| Button | Variants: primary, secondary, destructive, ghost. Height 40px default, 44px icon-only. Primary uses accent only. |
| Input/Select/Textarea | 40px height for input/select. Visible label required. Error text below field in 14px destructive. |
| Table | Header 14px/600. Body 14px or 16px. Row hover uses muted surface. No zebra striping unless needed for readability. |
| Badge | Use for role, workspace status, request status. Role colors must be subtle; do not imply access is secure by color alone. |
| Card | White surface, 1px border, 16–24px padding, 12px radius. |
| Dialog | Use only for confirmations and focused create/edit flows. Destructive dialog requires explicit action label. |
| Alert | Use for permission denied, backend errors, and audit warnings. Avoid toast-only critical feedback. |
| Empty state | Center within content card, 48px top/bottom padding, heading + body + optional CTA. |

## Copywriting Contract

All user-facing copy must be Vietnamese.

| Element | Copy |
|---------|------|
| Primary CTA | Tạo người dùng |
| Secondary CTA | Tạo workspace |
| Request shell CTA | Tạo hồ sơ yêu cầu |
| Empty state heading | Chưa có dữ liệu |
| Empty state body | Chưa có bản ghi nào trong phạm vi bạn được phép xem. Hãy tạo bản ghi mới hoặc kiểm tra quyền truy cập. |
| Error state | Không thể tải dữ liệu. Vui lòng thử lại hoặc liên hệ quản trị viên nếu lỗi tiếp diễn. |
| Permission denied | Bạn không có quyền xem nội dung này. Nếu cần truy cập, hãy liên hệ quản trị viên. |
| Workflow unavailable | Trạng thái này chỉ có thể thay đổi qua quy trình hợp lệ trên máy chủ. |
| Destructive confirmation | Vô hiệu hóa người dùng: Nhập xác nhận để vô hiệu hóa người dùng này. Hành động sẽ được ghi audit. |
| Destructive confirmation | Xóa thành viên khỏi workspace: Xác nhận xóa quyền truy cập workspace của người dùng này. Hành động sẽ được ghi audit. |
| Destructive confirmation | Hủy yêu cầu: Nhập lý do hủy yêu cầu. Hành động sẽ được ghi audit. |

Copy rules:

- Use verbs tied to domain action: `Tạo người dùng`, `Cập nhật vai trò`, `Chuyển trạng thái`, `Xem audit`.
- Never promise legal advice or AI decision.
- Error messages must not reveal whether inaccessible tenant/request exists.
- Audit copy must say action is recorded when user changes roles, memberships, assignment, or request status.

Source: CLAUDE.md requires Vietnamese user-facing exchange; Phase 1 FND requirements.
