## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge inner gaps, inline helper spacing |
| sm | 8px | Compact field hints, radio/card internal groups |
| md | 16px | Default element spacing, mobile card padding, form row gaps |
| lg | 24px | Section padding, desktop card padding, step block gaps |
| xl | 32px | Main layout gaps between service selection, questions, upload, review |
| 2xl | 48px | Major route sections and submitted status hero separation |
| 3xl | 64px | Page-level top/bottom spacing on desktop |

Exceptions: clickable controls must keep minimum 40px height from existing `Button`; icon-only controls, if any, must be 44px square for touch. File drop zone minimum height 160px. Progress step hit areas minimum 40px high.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#F8FAFC` | Page background, quiet empty areas, non-active progress rail |
| Secondary (30%) | `#FFFFFF` | Cards, service option surfaces, question panels, upload area, review summary |
| Accent (10%) | `#0F766E` | Primary CTA, selected service border/check, active progress step, focus ring, submitted success badge/icon |
| Destructive | `#DC2626` | Destructive actions only |

Accent reserved for: primary CTA (`Tiếp tục`, `Gửi yêu cầu`), selected service option, active/completed progress step, file upload focus state, submitted success/status highlight, focus ring. Never use accent for all links or all neutral icons.

Supporting colors from existing UI:
- Border: `#E2E8F0` default card/input border.
- Muted surface: `#F1F5F9` hover/secondary background.
- Secondary text: `#475569`.
- Warning/triage badge: amber `bg-amber-50`, text `#B45309`, border `amber-200`.
- Info badge: blue `bg-blue-50`, text `#1D4ED8`, border `blue-200`.

60/30/10 contract for Phase 2:
- Dominant: page shell and background gradient from `globals.css`.
- Secondary: all intake cards and form containers.
- Accent: completion/selection/action only, kept sparse for trust and clarity.

## Interaction Contract

| Area | Contract |
|------|----------|
| Route shape | Customer-facing route separate from admin, recommended `/intake` for start and `/requests/[requestId]` for read-only status. |
| Flow | Service selection -> guided questions -> upload area -> review/submit -> submitted status view. |
| Service selection | Render service options as clickable white cards with border. Selected card uses accent border/ring and visible selected state. Include unsupported option. |
| Progress | Show 4 steps: `Dịch vụ`, `Câu hỏi`, `Tài liệu`, `Kiểm tra`. Active/completed steps use accent; future steps use muted gray. |
| Guided chat/form | Use card-based prompts that feel conversational but save structured fields. One current question group per section; do not store raw chat only. |
| Required validation | Inline validation below fields in destructive color. Browser can guide, but backend remains source of truth. |
| Draft saving | Incomplete answers remain `draft_intake`; UI must not claim submission until backend transition succeeds. |
| Upload area | White card with dashed border `#CBD5E1`, accent border on focus/drag. Show selected/uploaded file list with filename and size only. No public URLs. |
| Review step | Show matter type, answer labels/values, and filenames before submit. Include privacy note. |
| Submit | Primary CTA `Gửi yêu cầu`; disabled only while submitting or when required visible fields are missing. Server validates again. |
| Submitted status | Read-only card with Vietnamese status label, next-step guidance, and no status mutation controls. |
| Unsupported path | Unsupported service can be submitted; status view shows triage warning badge and human classification guidance. |
| Destructive action | Only removing an attached file before submit is destructive. Require inline confirmation or confirm dialog with specified copy. |
| Keyboard/focus | All cards used as controls must be keyboard reachable. Focus ring uses `#0F766E` and 2px ring with offset. |
