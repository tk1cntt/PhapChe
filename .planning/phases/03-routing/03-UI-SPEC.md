## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge internal gaps, compact inline separation |
| sm | 8px | Label/value gaps, stacked form controls inside assignment forms |
| md | 16px | Default card padding on mobile, table cell horizontal padding, row action gaps |
| lg | 24px | Card padding on desktop, `PageHeader` padding, section blocks |
| xl | 32px | Page-level grid gaps between routing queue, matter admin, and capability admin sections |
| 2xl | 48px | Major separation between coordinator workspace groups when stacked |
| 3xl | 64px | Reserved for page-level empty state vertical breathing room only |

Exceptions: interactive controls must keep minimum 40px height via existing `Button min-h-10`; icon-only controls are not in Phase 3 scope, but if added later must be at least 44px touch target.

Interaction layout contract:
- Coordinator routing queue rows use 16px table cell padding and never rely on color alone for status or suggestion meaning.
- Assignment/reassignment reason field sits directly before submit action with 8px label/control gap and 16px gap before buttons.
- Specialist assigned queue uses same table density as admin requests; detail page uses 24px card padding and 32px section gaps.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 | 1.5 |
| Label | 14px | 600 | 1.4 |
| Heading | 20px | 600 | 1.2 |
| Display | 30px | 600 | 1.15 |

Typography contract:
- Use exactly two weights in new Phase 3 UI: 400 normal and 600 semibold.
- Use 13px only for existing `Badge` component because it already ships in admin UI; do not introduce more text sizes.
- Table headers keep existing 12px uppercase tracking from `Table`; do not create custom header style.
- Request IDs, matter keys, and enum-like technical values may use body size with muted secondary color; do not use monospace unless existing component requires it.

Sources: `src/app/admin/components/ui.tsx` uses 30px page title, 16px description, 14px buttons, 13px badges, 12px table headers; `src/app/globals.css` sets body line-height 1.5.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #F8FAFC | Page background, table header background, muted empty-state blocks |
| Secondary (30%) | #FFFFFF | Cards, page header surface, tables, forms, specialist detail panels |
| Accent (10%) | #0F766E | Primary assignment CTA, focus rings, selected/suggested candidate badge, safe success/accent badge only |
| Destructive | #DC2626 | Destructive actions only: deactivating matter type/capability if present; cancelling/discarding assignment draft if implemented as destructive |

Accent reserved for: `Phân công chuyên viên`, `Cập nhật phân công`, selected eligible candidate indicator, focus ring, active routing/matter/capability section indicator, non-destructive success state after assignment.

Semantic supporting colors from existing admin badges:
- Info blue `#1D4ED8`: reviewer suggestion badge or informational helper only.
- Warning amber `#B45309`: triage/submitted attention states and missing capability warning.
- Neutral slate `#475569`: default badges, metadata, secondary text.
- Red `#B91C1C` badge text with `#DC2626` destructive action: failed validation or destructive controls only.

Color rules:
- Never use accent for every link/button; secondary buttons stay white/slate as existing `Button secondary`.
- Status must include Vietnamese text label, not color-only cue.
- Assignment eligibility reason must be text visible in table/card, not hidden behind color.

Sources: `src/app/globals.css` declares `--background`, `--surface`, `--accent`, `--destructive`, `--text-primary`, `--text-secondary`, `--border`, `--muted`; `src/app/admin/components/ui.tsx` declares button and badge colors.

## Interaction Contract

### Coordinator/Admin routing queue
- Default scope: show submitted/triage requests eligible for routing plus already assigned requests only when reassignment action is available in this phase.
- Required columns: Yêu cầu, Khách hàng, Loại vụ việc, Trạng thái, Chuyên viên hiện tại, Reviewer hiện tại, Gợi ý chuyên viên, Gợi ý reviewer, Hành động.
- Each suggestion must show person name/email and reason text. Minimum reason: `Phù hợp vai trò và năng lực với loại vụ việc này.`
- Coordinator/admin must explicitly choose assignee. UI must not auto-select first eligible person as final action.
- Assignment and reassignment must require non-empty reason before submit.
- Submit state: disable primary button and show pending text `Đang lưu...` while server action runs.
- Failure state: keep entered reason and selected assignee visible; show error copy near form.
- Success state: refresh row assignee/status from server response; show success copy.
- Frontend may display current status and allowed actions, but must not hard-code status mutation. Server action owns transition.

### Matter type admin
- Use card/table form language, no new design system.
- Required fields visible: Tên loại vụ việc, Mã loại vụ việc, Mô tả, Trạng thái.
- Matter key remains English/stable; label and description shown in Vietnamese.
- Inactive matter types use neutral or warning badge, not destructive red unless deletion/destructive action exists.

### Capability admin
- Required fields visible: Người dùng, Vai trò năng lực, Loại vụ việc, Trạng thái.
- Only active workspace members should appear in add/edit controls for MVP.
- Specialist and reviewer capabilities must be visually separated by label/badge or grouped sections.
- Missing capability state must explain next step: configure capability before suggestions appear.

### Specialist assigned queue/details
- Queue shows only server-authorized requests assigned to current specialist.
- Required queue columns: Yêu cầu, Khách hàng, Loại vụ việc, Trạng thái, Ngày gửi, Hành động.
- Detail page shows intake summary and supporting file metadata only.
- No document drafting controls, reviewer QC controls, or final delivery controls.
- File metadata display may show file name, type, size, uploaded time if available; do not add download/preview behavior beyond existing authorized patterns.

### Accessibility and usability
- All inputs have visible Vietnamese labels.
- Focus ring uses `#0F766E` and existing `focus:ring-2 focus:ring-offset-2` pattern.
- Buttons must have disabled styling from existing component and remain readable at disabled opacity.
- Tables must remain horizontally scrollable using existing `Table` wrapper.
- Errors must be text adjacent to the relevant form, not toast-only.
