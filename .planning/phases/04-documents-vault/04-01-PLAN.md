## Schema design decisions

Per D-01, D-02, D-03:
- `DocumentTemplate` model with workspaceId + matterTypeKey + version as unique key
- `status` enum: draft, approved, published, deprecated
- `previousVersionId` optional self-relation for version lineage
- `variableSchema` JSON: array of { key, label, required, type }
- Only approved or published templates can be used for generation (enforced in service)

## RBAC requirements

Per D-14 and FND-02:
- Template list: coordinator_admin, super_admin only
- Template create/update/publish: coordinator_admin, super_admin only
- Template read (for generation): specialist also needs read access (list by workspace + matter type)

<task type="auto">
  <name>Task 2: Create template service layer</name>
  <files>src/lib/documents/template-service.ts, src/lib/documents/template-service.test.ts</files>
  <read_first>src/lib/intake/upload-service.ts, src/lib/workflow/request-workflow.ts</read_first>
  <action>
    Create src/lib/documents/template-service.ts with these functions:

    **listTemplates(session, workspaceId, matterTypeKey?)**
    - Check workspace access with canAccessWorkspace
    - Return templates filtered by workspaceId, optionally by matterTypeKey
    - Include previousVersionId for version chain display
    - Order by matterTypeKey, then version desc

    **createTemplate(session, input)**
    - input: { workspaceId, matterTypeKey, label, description?, variableSchema, content }
    - Require coordinator_admin or super_admin role
    - Set version to 1, status to draft
    - Return created template

    **updateTemplate(session, templateId, input)**
    - Load existing template
    - Require coordinator_admin or super_admin
    - If status is published or deprecated, throw 'TEMPLATE_IMMUTABLE'
    - Update allowed fields (label, description, variableSchema, content)
    - Return updated template

    **publishTemplate(session, templateId)**
    - Load existing template
    - Require coordinator_admin or super_admin
    - If status is published or deprecated, throw 'TEMPLATE_ALREADY_PUBLISHED'
    - Set status to published
    - Audit event: action='template.published', metadataSummary includes templateId, matterTypeKey, version
    - Return updated template

    **approveTemplate(session, templateId)**
    - Similar to publish but set status to approved instead
    - Audit event: action='template.approved'

    **deprecateTemplate(session, templateId)**
    - Load existing template
    - Require coordinator_admin or super_admin
    - Set status to deprecated
    - Audit event: action='template.deprecated'

    **createNewVersion(session, templateId, input)**
    - Load existing published/approved template
    - Create NEW version with version = existing.version + 1
    - Set previousVersionId to old template's id
    - Set status to draft (new version starts as draft)
    - Copy over matterTypeKey from original
    - Audit event: action='template.version_created', metadataSummary includes newVersionId, oldVersionId, matterTypeKey
    - Return new template (draft status, editable)

    **getTemplatesForGeneration(session, workspaceId, matterTypeKey)**
    - List templates where workspaceId=matterTypeKey matches AND status in [approved, published]
    - Used by specialist workbench to show available templates
    - Only return latest version per matterTypeKey (hide older superseded versions)

    Variable schema format (variableSchema JSON):
    ```json
    [
      {"key": "company_name", "label": "Tên công ty", "required": true, "type": "text"},
      {"key": "contract_value", "label": "Giá trị hợp đồng", "required": false, "type": "number"},
      {"key": "effective_date", "label": "Ngày có hiệu lực", "required": true, "type": "date"}
    ]
    ```
  </action>
