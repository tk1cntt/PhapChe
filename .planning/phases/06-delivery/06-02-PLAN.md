## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> `/api/vault/[vaultFileId]/download` | Untrusted `vaultFileId` and query params. |
| Download route -> vault service | Session and file identifier checked before signed URL/response. |
| Vault service -> audit log | Sensitive legal file context reduced to safe identifiers. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-02-01 | Elevation of Privilege | Download route | mitigate | Require session, `canAccessVaultFile`, customer ownership/workspace, and final document linkage. |
| T-06-02-02 | Information Disclosure | `requestVaultFileAccess` | mitigate | Do not return or log raw `storageKey`; return server-mediated URL and safe metadata only. |
| T-06-02-03 | Information Disclosure | Download response cache | mitigate | Set `Cache-Control: no-store`; TTL 15 minutes; expired response 410. |
| T-06-02-04 | Repudiation | Vault access audit | mitigate | Record actorId, vaultFileId, requestId, expiry timestamp via `recordAuditEvent`. |
</threat_model>
