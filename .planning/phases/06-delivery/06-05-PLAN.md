<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Customer browser -> download route | Untrusted query params attempt to access legal file. |
| Download route -> vault service | Server verifies session/RBAC/final document before reading private storage. |
| Vault service -> audit log | Sensitive legal file metadata crosses logging boundary. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-05-01 | Tampering | Signed URL query | mitigate | HMAC signs exact `vaultFileId.userId.expires`; route verifies with `timingSafeEqual`; edited `expires`, `userId`, or `vaultFileId` returns 403. |
| T-06-05-02 | Information Disclosure | Download route | mitigate | `requireAppSession`, `canAccessVaultFile`, customer final-document check in `getVaultFileDownloadPayload`, no storageKey in response. |
| T-06-05-03 | Elevation of Privilege | Cross-user signed link reuse | mitigate | Signed payload includes `userId`; route requires query `userId === session.userId` before file read. |
| T-06-05-04 | Repudiation | Access request audit | mitigate | `requestVaultFileAccess` records safe audit metadata with actorId, vaultFileId, requestId, expiresAt; no legal content/raw storageKey. |
| T-06-05-05 | Denial of Service | Malformed signature | mitigate | Verification returns false for malformed hex without throwing; route returns controlled 403. |
</threat_model>
