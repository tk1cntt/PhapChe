<threat_model>
## Trust Boundaries
| Boundary | Description |
|---|---|
| Admin route/service caller -> ops service | Caller-provided session, workspace, request, and filter params cross into privileged ops data access. |
| Ops service -> Prisma | Normalized filter params become database predicates over tenant legal request data. |
| Audit/workflow rows -> UI DTO | Sensitive operational records are transformed into display-safe timeline rows. |

## STRIDE Threat Register
| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-07-01-01 | Information Disclosure | `requireOpsAdmin` | mitigate | Query active `workspaceMembership.findFirst` for `coordinator_admin` or `super_admin` before returning any ops DTO; throw `FORBIDDEN` before metric/timeline queries. |
| T-07-01-02 | Tampering | `parseOpsFilters` | mitigate | Normalize and allowlist request status/date/user/matter filters; build typed Prisma `where`, never raw SQL. |
| T-07-01-03 | Information Disclosure | timeline DTO | mitigate | Select and return safe fields only; never select generated content, reviewer comments, raw intake answers, storage keys, or file contents. |
| T-07-01-04 | Integrity | metrics queries | mitigate | Use one shared normalized filter builder for dashboard counts and request list so counts/list cannot diverge. |
</threat_model>
