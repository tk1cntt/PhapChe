<threat_model>
## Trust Boundaries
| Boundary | Description |
|---|---|
| Admin route/service caller -> ops service | Caller-provided workspace, user, query params cross into privileged ops data access. |
| Ops service -> Prisma | Filter params become database predicates over tenant legal request data. |
| Audit/workflow rows -> UI DTO | Sensitive operational records are transformed into display-safe timeline rows. |

## STRIDE Threat Register
| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-07-01-01 | Information Disclosure | `requireOpsAdmin` | High | mitigate/block | Query `workspaceMembership.findFirst` for active `coordinator_admin` or `super_admin` before returning any ops DTO; throw `FORBIDDEN` before metric/timeline queries. |
| T-07-01-02 | Tampering | filter parser | High | mitigate/block | Allowlist `RequestStatus`, user IDs, workspace IDs, matter type keys, and parse valid date range; reject invalid dates/status rather than interpolating raw params. |
| T-07-01-03 | Information Disclosure | timeline DTO | High | mitigate/block | Select only `createdAt`, actor safe label, action/status, target identifier, correlationId, reason, `metadataSummary`; do not select `generatedContent`, `generalComment`, `storageKey`, raw intake answers, file contents. |
| T-07-01-04 | Integrity | metrics queries | Medium | mitigate | Use one normalized filter builder for dashboard counts and request list so counts/list cannot diverge. |
</threat_model>
