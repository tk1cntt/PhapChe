<threat_model>
## Trust Boundaries
| Boundary | Description |
|---|---|
| Browser query string -> server page | User-controlled filters enter server-rendered page. |
| Server page -> ops service | Page requests privileged operational DTOs. |
| Ops DTO -> HTML | Sensitive backend data is rendered to admin UI. |

## STRIDE Threat Register
| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-07-02-01 | Information Disclosure | `/admin/ops/page.tsx` | High | mitigate/block | Always call `requireAppSession`, then `parseOpsFilters`/`getOpsDashboard`; do not query Prisma directly in UI page and do not bypass `requireOpsAdmin`. |
| T-07-02-02 | Tampering | filter form/query params | High | mitigate/block | Submit filters through URL params but trust only service-normalized filter DTO; preserve AND semantics and render error state for invalid filters. |
| T-07-02-03 | Integrity | dashboard/list metrics | High | mitigate/block | Render all counts/list/workload from the single `getOpsDashboard` payload; no hard-coded fixtures and no client-only recalculation. |
| T-07-02-04 | Information Disclosure | request table cells | Medium | mitigate | Render identifiers, labels, statuses, assignees, ages only; do not add columns for raw intake answers, document text, reviewer comments, vault keys, or file content. |
</threat_model>
