    Read the existing schema carefully. Make these changes:

    1. **Add to existing User model** (before `createdAt`):

    2. **Add to existing User model** (after `vaultFiles VaultFile[]` line, before the closing `}`):
       ```

       ```

    3. **Add three new models** at the end of the file (after the existing `model AuditEvent` closing `}`, before any trailing comments):

    ```prisma
    model Account {
      id                    String    @id @default(cuid())
      userId                String
      accountId             String
      providerId            String
      accessToken           String?
      refreshToken          String?
      accessTokenExpiresAt  DateTime?
      refreshTokenExpiresAt DateTime?
      scope                 String?
      idToken               String?
      password              String?
      createdAt             DateTime  @default(now())
      updatedAt             DateTime  @updatedAt

      user User @relation(fields: [userId], references: [id], onDelete: Cascade)

      @@unique([providerId, accountId])
    }

    model Session {
      id        String   @id @default(cuid())
      userId    String
      token     String   @unique
      expiresAt DateTime
      ipAddress String?
      userAgent String?
      createdAt DateTime @default(now())
      updatedAt DateTime @updatedAt

      user User @relation(fields: [userId], references: [id], onDelete: Cascade)
    }

    model Verification {
      id         String   @id @default(cuid())
      identifier String
      value      String
      expiresAt  DateTime
      createdAt  DateTime @default(now())
      updatedAt  DateTime @updatedAt
    }
    ```

    IMPORTANT: Preserve ALL existing models, fields, enums, relations, and indexes unchanged. The User model must retain `memberships WorkspaceMembership[]`, `createdRequests`, `specialistRequests`, and all other existing relations.

<task type="auto">
  <name>Task 3: Push schema to database and regenerate Prisma client</name>
  <files>prisma/schema.prisma</files>
  <read_first>@prisma/schema.prisma</read_first>
  <action>
    Run these commands in order:

    1. `npx prisma db push --accept-data-loss`
       - This creates the new Account, Session, Verification tables and adds emailVerified column to User
       - `--accept-data-loss` is required because this is non-TTY mode (no data will actually be lost — we're adding tables/columns, not removing)

    2. `npx prisma generate`
       - Regenerates the Prisma client so the new models are available in TypeScript

    If `npx prisma db push` fails due to database connection issues, output the error for diagnosis but do NOT block — the schema file changes are still correct and can be pushed later when DB is available.
  </action>
  <acceptance_criteria>
    - `npx prisma db push --accept-data-loss` exits with 0
    - `npx prisma generate` exits with 0
    - New Prisma client types include Account, Session, Verification
  </acceptance_criteria>
  <verify>
    npx prisma db push --accept-data-loss 2>&1 && npx prisma generate 2>&1
  </verify>
  <done>
    Database has Account/Session/Verification tables; Prisma client regenerated with new model types
