<objective>
Fix the runtime failures caused by Ant Design table render functions and invalid element rendering on queue/ops pages while preserving server-side data fetching and authorization.
</objective>

<must_haves>
- `/specialist/requests` renders without `Functions cannot be passed directly to Client Components`.
- `/reviewer/requests` renders without `Functions cannot be passed directly to Client Components`.
- `/admin/ops` renders without `Element type is invalid`.
- `/admin/ops/[requestId]` renders without `Element type is invalid`.
- Prisma queries and `requireAppSession()` remain in server components/services.
</must_haves>

<threat_model>
- **Threat:** Moving protected server logic into client components exposes data or bypasses checks. **Mitigation:** client components receive only already-authorized serializable DTOs.
- **Threat:** Ops/audit pages expose raw legal content. **Mitigation:** preserve existing ops service safe fields: identifiers, status/action, reason, correlation ID, metadataSummary.
</threat_model>
