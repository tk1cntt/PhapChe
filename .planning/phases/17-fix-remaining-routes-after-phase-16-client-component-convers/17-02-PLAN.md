## Phase 16 Harness loadFixtures() Function (lines 74-121)

```javascript
async function loadFixtures() {
  const fixtures = {
    samples: { requestId: 'sample-request-id', templateId: 'sample-template-id', documentVersionId: 'sample-document-version-id' },
    ...
  };
  try {
    const { PrismaClient } = require('@prisma/client');
    const prisma = new PrismaClient();
    const workspace = await prisma.workspace.findFirst({ select: { id: true, slug: true } });
    if (workspace) fixtures.workspace = workspace;

    const request = await prisma.legalRequest.findFirst({
      where: { workspaceId: workspace?.id },
      orderBy: { createdAt: 'desc' },
      select: { id: true },
    });
    if (request?.id) fixtures.samples.requestId = request.id;
    // ... template and docVersion queries follow
  } catch (error) {
    fixtures.dbError = String(error?.message ?? error);
  }
  return fixtures;
}
```

The code looks correct but the output shows placeholder values. The `dbError` field is not present, meaning no exception was thrown. The issue may be that `workspace?.id` is being used but `workspace` might be from a different workspace or the queries are returning null.
