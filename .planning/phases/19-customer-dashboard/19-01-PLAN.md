# Plan 19-01 — Customer Request Dashboard

<objective>
Build `/customer/requests` so a logged-in customer can see only their own legal requests in the active workspace, open status detail for every request, open delivered results only for delivered/closed requests, and navigate back to the dashboard from `/requests/[requestId]`.
</objective>

<threat_model>

| Threat ID | Severity | Threat | Mitigation in this plan | Verification |
|---|---|---|---|---|
| T-19-01 | high | Cross-tenant request leak on customer dashboard | Query `LegalRequest` server-side with `workspaceId: session.activeWorkspaceId ?? ''` | Read `src/app/customer/requests/page.tsx` and confirm `workspaceId` filter exists |
| T-19-02 | high | Cross-user request leak within same workspace | Query `LegalRequest` server-side with `createdById: session.userId` | Read `src/app/customer/requests/page.tsx` and confirm `createdById: session.userId` exists |
| T-19-03 | high | Premature final-document exposure for non-delivered requests | Render `Xem tài liệu` only when `status === 'delivered' || status === 'closed'`; keep download flow on existing `/customer/requests/[requestId]` page | E2E asserts non-delivered intake request does not show `Xem tài liệu` |
| T-19-04 | medium | Internal metadata disclosure to customers | Select/render only `id`, `title`, `status`, `createdAt`, `updatedAt`, and `intakeSubmission.matterTypeKey` | Read selected fields in `page.tsx` and rendered columns in table |
| T-19-05 | medium | Frontend-only filtering accidentally returns unauthorized rows | Do not create a broad API list; enforce owner/workspace scoping in server component before passing rows to client table | Read `page.tsx` query and absence of dashboard API route in plan scope |

</threat_model>

<must_haves>
- `/customer/requests` route exists and renders without blank page/runtime component errors.
- Dashboard data is scoped by both `workspaceId` and `createdById` using `requireAppSession()` server-side.
- Dashboard shows title, Vietnamese status tag, matter type fallback, created date, and updated date.
- Every row links to `/requests/[requestId]` with `Xem trạng thái`.
- Only `delivered`/`closed` rows link to `/customer/requests/[requestId]` with `Xem tài liệu`.
- Empty state links to `/intake` with customer-friendly Vietnamese copy.
- `/requests/[requestId]` includes a CTA back to `/customer/requests`.
- Playwright E2E covers the new UI feature.
</must_haves>

<action>
Create `src/app/customer/requests/page.tsx` as an async server component that imports `Link` from `next/link`, `Card`, `Button`, `Flex`, and `Typography` from `antd`, `prisma` from `@/lib/prisma`, `requireAppSession` from `@/lib/security/session`, and `CustomerRequestsTable` from `./CustomerRequestsTable`.

In `page.tsx`, call `const session = await requireAppSession();` before reading data. Query:

```ts
const requests = await prisma.legalRequest.findMany({
  where: {
    workspaceId: session.activeWorkspaceId ?? '',
    createdById: session.userId,
  },
  select: {
    id: true,
    title: true,
    status: true,
    createdAt: true,
    updatedAt: true,
    intakeSubmission: { select: { matterTypeKey: true } },
  },
  orderBy: { createdAt: 'desc' },
});
```

Serialize rows with `createdAt: request.createdAt.toISOString()`, `updatedAt: request.updatedAt.toISOString()`, and `matterTypeKey: request.intakeSubmission?.matterTypeKey ?? null`.

Render within a max-width `960` container under the existing customer layout. Header copy must be:
- H1: `Yêu cầu của tôi`
- Body: `Theo dõi các yêu cầu pháp lý bạn đã gửi và mở tài liệu cuối cùng khi hồ sơ đã được giao.`
- CTA button text: `Tạo yêu cầu mới`, href `/intake`

If `rows.length === 0`, render an empty state with heading `Bạn chưa có yêu cầu nào`, body `Hãy gửi yêu cầu pháp lý đầu tiên để đội ngũ Pháp Chế bắt đầu tiếp nhận và xử lý.`, and `/intake` CTA `Tạo yêu cầu mới`.

Create `src/app/customer/requests/CustomerRequestsTable.tsx` as a client component. It must import `Link`, `ColumnsType`, `Table`, `Tag`, `Button`, and `RequestStatus`. Define a `CustomerRequestRow` type with `id`, `title`, `status`, `createdAt`, `updatedAt`, and `matterTypeKey`.

Copy the Vietnamese status label/tone mapping from existing request pages:
- `draft_intake`: `Nháp tiếp nhận`, neutral
- `intake_submitted`: `Đã gửi tiếp nhận`, info
- `triage`: `Đang phân loại`, warning
- `assigned`: `Đã phân công`, info
- `in_progress`: `Đang xử lý`, info
- `pending_review`: `Chờ kiểm tra chất lượng`, warning
- `revision_required`: `Cần chỉnh sửa nội bộ`, warning
- `approved`: `Đã được duyệt`, accent
- `delivered`: `Đã giao tài liệu`, outline
- `closed`: `Đã đóng hồ sơ`, neutral
- `cancelled`: `Đã hủy`, destructive

Map tones to Ant Design colors exactly: neutral/default, info/blue, warning/orange, accent/cyan, destructive/red, outline/default.

Render table columns:
- `Yêu cầu`: title styled with `#0F172A`, `fontWeight: 400`, `fontSize: 16`.
- `Loại vụ việc`: `matterTypeKey ?? 'Chưa có loại vụ việc'`.
- `Trạng thái`: status `Tag`.
- `Ngày tạo`: `formatDate(record.createdAt)` using `new Intl.DateTimeFormat('vi-VN', { dateStyle: 'medium' })`.
- `Cập nhật`: `formatDate(record.updatedAt)`.
- `Hành động`: a `Flex`/inline action group with `Xem trạng thái` linking to `/requests/${record.id}` for every row, and `Xem tài liệu` linking to `/customer/requests/${record.id}` only when `record.status === 'delivered' || record.status === 'closed'`.

Set `rowKey="id"`, `pagination={false}`, and `locale={{ emptyText: 'Bạn chưa có yêu cầu nào.' }}`.
</action>
