## Objective

Connect user dashboard (`src/app/[locale]/dashboard/page.tsx`) to real Prisma queries. Replace placeholder "Dashboard coming soon" with full dashboard matching `layout/user-dashboard.html`.

## Tasks

### Task 1: Create Dashboard API Endpoint

<read_first>
- src/app/api/admin/requests/route.ts (admin pattern reference)
- src/lib/security/session.ts (requireAppSession usage)
</read_first>

<action>
Create `src/app/api/dashboard/route.ts` with GET handler:

1. Call `requireAppSession()` to get session with userId/workspaceId
2. Execute parallel Prisma queries:
   - `prisma.legalRequest.count({ where: { workspaceId } })` → totalRequests
   - `prisma.legalRequest.count({ where: { workspaceId, status: { in: ['in_progress', 'pending_review', 'assigned'] } } })` → inProgressCount
   - `prisma.legalRequest.count({ where: { workspaceId, status: { in: ['approved', 'delivered', 'closed'] } } })` → completedCount
   - `prisma.vaultFile.count({ where: { workspaceId } })` → vaultDocsCount
   - `prisma.legalRequest.findMany({ where: { workspaceId, status: { in: ['in_progress', 'pending_review', 'revision_required'] } }, take: 5, orderBy: { updatedAt: 'desc' }, include: { assignedSpecialist: { select: { name: true } }, assignedReviewer: { select: { name: true } } } })` → recentCases
   - `prisma.legalRequest.findMany({ where: { workspaceId, slaDeadline: { gte: subDays(now, 1), lte: addDays(now, 7) } } }, take: 5, orderBy: { slaDeadline: 'asc' })` → deadlines
   - `prisma.vaultFile.findMany({ where: { workspaceId }, take: 5, orderBy: { updatedAt: 'desc' }, include: { uploadedBy: { select: { name: true } } } })` → recentDocs
   - `prisma.auditLog.findMany({ where: { workspaceId }, take: 10, orderBy: { createdAt: 'desc' } })` → activityLog

3. Return JSON with all data for dashboard rendering
</action>

<acceptance_criteria>
- GET /api/dashboard returns 200 with dashboard stats
- Response includes: totalRequests, inProgressCount, completedCount, vaultDocsCount, recentCases, deadlines, recentDocs, activityLog
- API requires authenticated session (403 if not logged in)
- Prisma queries are workspace-scoped (workspaceId filter)
</acceptance_criteria>
