<objective>
Implement Local-first Storage Infrastructure with S3-ready architecture. All file operations flow through StorageService abstraction, file metadata stored in database, ObjectKey convention enables seamless migration to S3.

**Purpose:** Provide pluggable storage backend (local for MVP, S3 for production) without changing business logic.

**Output:** StorageProvider interface, LocalStorageAdapter, StorageService, File API routes, database schema, audit logging, migration command skeleton.
</objective>

         - Verify user has delete permission
         - Call provider.deleteObject()
         - Update File.status to 'deleted'
         - Call recordFileAccessLog(actorId, fileId, 'delete')
         - Return success
       
       - getAccessLogs(fileId, pagination):
         - Fetch FileAccessLog records for fileId
         - Return paginated results with user info

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → API | Untrusted file uploads cross here |
| API → StorageService | Authenticated operations |
| StorageService → LocalStorageAdapter | Trusted internal calls |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-56-01 | Tampering | File upload | mitigate | Validate MIME type whitelist, file size limit (50MB), checksum verification |
| T-56-02 | Information Disclosure | File download | mitigate | Permission check (organization membership), private storage path not accessible via static serve |
| T-56-03 | Denial of Service | File upload | mitigate | Rate limiting on upload endpoint, file size limit, storage quota per organization (future) |
| T-56-04 | Tampering | ObjectKey | mitigate | Backend generates objectKey, never trust user input, sanitize filename |
| T-56-05 | Elevation of Privilege | File access | mitigate | Role-based permission check in StorageService, FileAccessLog audit trail |
| T-56-06 | Information Disclosure | Storage path | mitigate | Local path /data/storage/private NOT in public/, backend generates signed URLs |
| T-56-SC | Supply Chain | npm packages | accept | Only using Node.js fs/crypto builtins, no new storage-specific packages |
</threat_model>
