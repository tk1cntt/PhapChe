# Phase 68 Wave 2: Partner Auth UI

## Objective

Implement UI pages and components for partner member management - invite system, member list, and accept invite flow.

## Context

Phase 66 Wave 2 implemented partner auth APIs. Now we need UI to let partner admins:
- Invite new team members via email
- View and manage existing members
- Accept/decline invite links

## Tasks

### Task 1: Partner Settings Page Structure
**Files:** `src/app/[locale]/partner/settings/page.tsx`, `src/components/partner/PartnerSettingsClient.tsx`

Create partner settings page at `/vi/partner/settings`:
- Tab navigation: Members, Invites, Profile
- Role badge showing current user's role
- Permission indicators

### Task 3: Members Table Component
**Files:** `src/components/partner/MembersTable.tsx`

Create table component displaying team members:
- Columns: Avatar, Name, Email, Role, Status, Joined, Actions
- Role badge with color coding (admin=purple, specialist=blue, viewer=gray)
- Status indicator (active=green, inactive=gray)
- Actions dropdown: Change Role, Deactivate, Remove
- Empty state when no members

```typescript
interface Member {
  id: string;
  user: { id: string; name: string; email: string; avatarUrl?: string };
  role: 'admin' | 'specialist' | 'viewer';
  isActive: boolean;
  joinedAt: string;
}
```

**API Call:**
```typescript
GET /api/partner/members
```

**Verify:** Table displays all members with correct data

### Task 4: Member Actions Dropdown
**Files:** `src/components/partner/MemberActions.tsx`

Create dropdown for member management:
- Change Role option (opens confirmation)
- Deactivate/Activate toggle
- Remove Member option (with confirmation)
- Only admins can see these actions

```typescript
interface MemberActionsProps {

```

**API Calls:**

**Verify:** Dropdown shows correct options based on role, actions work

## API Endpoints Used

| Method | Endpoint | Purpose |
|--------|----------|---------|
| GET | `/api/partner/members` | List members |
| PATCH | `/api/partner/members/[id]` | Update member |
| DELETE | `/api/partner/members/[id]` | Remove member |
| POST | `/api/partner/invite` | Create invite |
| GET | `/api/partner/invite` | List invites |
| DELETE | `/api/partner/invite/[id]` | Revoke invite |
| POST | `/api/partner/invite/accept` | Accept invite |
