# Plan 02: API Client + React Query Foundation

## Objective

Enhance existing ApiClient với auto-retry, toast notifications, 401 redirect. Tạo centralized React Query setup với QueryClientProvider, query key factories, và domain hooks. Refactor existing hooks to use ApiClient.

## Tasks

### Task 2.1: Enhance ApiClient with retry logic

<read_first>
- src/lib/api/client.ts (existing API client — must enhance NON-BREAKING)
- .planning/phases/73-shared-foundation/73-CONTEXT.md (Decision 2: API client conventions)
- .planning/phases/73-shared-foundation/73-SPEC.md (R3: auto-retry 3x, R4: typed methods)
</read_first>

<action>
Enhance `src/lib/api/client.ts` ApiClient.request() method:
1. Add retry loop around `fetch()` call (max 3 retries)
2. Retry ONLY on: network errors (`TypeError: Failed to fetch`) and HTTP 502/503/504
3. Delays: 500ms → 1000ms → 2000ms (exponential backoff)
4. Use `await new Promise(resolve => setTimeout(resolve, delay))` for delays
5. Do NOT retry on 4xx errors (client errors)
6. Keep existing method signatures: `get<T>`, `post<T>`, `put<T>`, `patch<T>`, `delete<T>` unchanged
7. Existing typed methods already satisfy FOUND-04 — verify and document
</action>

<acceptance_criteria>
- `src/lib/api/client.ts` contains retry logic with 3 attempts
- Retry delays are 500, 1000, 2000 ms
- Only retries on network errors and 502/503/504 responses
- Does NOT retry on 400, 401, 403, 404, 500 responses
- Existing `get<T>`, `post<T>`, `put<T>`, `patch<T>`, `delete<T>` signatures unchanged
- All existing imports from `src/lib/api/index.ts` continue working
</acceptance_criteria>

<verify>
- `npx tsc --noEmit src/lib/api/client.ts` — compiles
- `grep 'retry' src/lib/api/client.ts` — retry logic present
- `grep 'setTimeout' src/lib/api/client.ts` — backoff delays present
</verify>

### Task 2.2: Add toast notifications and 401 redirect

<read_first>
- src/lib/api/client.ts (after Task 2.1 changes)
- package.json (check if react-hot-toast is installed)
- .planning/phases/73-shared-foundation/73-CONTEXT.md (Decision 2: toast via react-hot-toast)
</read_first>

<action>
1. Install `react-hot-toast` if not already in package.json: `npm install react-hot-toast`
2. Create `src/lib/toast.ts` — thin wrapper around react-hot-toast:
   - Export `toastSuccess(message: string)`, `toastError(message: string)`, `toastInfo(message: string)`, `toastWarning(message: string)`
   - All use `toast()` from react-hot-toast with appropriate type
   - Position: top-right (configured in Toaster component)
3. Enhance ApiClient.request() error handling:
   - On HTTP 500: call `toastError('Lỗi máy chủ, vui lòng thử lại')`
   - On HTTP 403: call `toastError('Không có quyền truy cập')`
   - On HTTP 401 AND `typeof window !== 'undefined'`: redirect to `/login` via `window.location.href`
   - Toast calls only execute on client-side (guard with `typeof window !== 'undefined'`)
4. Keep error throwing behavior — toast is ADDITIONAL, not replacement for error propagation
</action>

- Toast functions guarded by `typeof window !== 'undefined'` check
- Original errors still thrown after toast (non-breaking)
- `react-hot-toast` in package.json dependencies
</acceptance_criteria>

### Task 2.3: Create centralized React Query setup

<read_first>
- package.json (verify @tanstack/react-query version — should be v5.x)
- src/app/layout.tsx (existing layout to add QueryClientProvider)
- .planning/phases/73-shared-foundation/73-SPEC.md (R5: QueryClient with defaults)
</read_first>

<action>
1. Create `src/lib/react-query.tsx`:
   - Import `QueryClient` from `@tanstack/react-query`
   - Export `queryClient` singleton with defaults:
     - `staleTime: 5 * 60 * 1000` (5 minutes)
     - `gcTime: 30 * 60 * 1000` (30 minutes — v5 naming, NOT cacheTime)
     - `retry: 3`
     - `refetchOnWindowFocus: false`
   - Export `QueryProvider` component that wraps children with `<QueryClientProvider client={queryClient}>`
   - Include `ReactQueryDevtools` in provider (dev only, `import { ReactQueryDevtools } from '@tanstack/react-query-devtools'`)

2. Create `src/lib/query-keys.ts` — query key factory:
   - `users.all`, `users.lists()`, `users.list(filters)`, `users.details()`, `users.detail(id)`
   - `requests.all`, `requests.lists()`, `requests.list(filters)`, `requests.details()`, `requests.detail(id)`
   - `workspaces` — same pattern
   - `auditEvents` — same pattern
   - `vaultFiles` — same pattern
   - `messages` — same pattern
   - All keys use array format: `['entity', { filters }]` or `['entity', 'detail', id]`

3. Update `src/app/layout.tsx`:
   - Import `QueryProvider` from `@/lib/react-query`
   - Import `Toaster` from `react-hot-toast`
   - Wrap app content with `<QueryProvider>` and add `<Toaster position="top-right" />`
   - Preserve existing providers and layout structure
</action>

<acceptance_criteria>
- `src/lib/react-query.tsx` exists with `queryClient` and `QueryProvider` exports
- `queryClient` uses `gcTime` (not `cacheTime`) — TanStack Query v5
- `staleTime` is 5 minutes (300000ms), `gcTime` is 30 minutes (1800000ms)
- `src/lib/query-keys.ts` exists with factories for all 6 domains
- All query keys use array format
- `src/app/layout.tsx` contains `<QueryProvider>` wrapper
- `src/app/layout.tsx` contains `<Toaster position="top-right" />`
</acceptance_criteria>
