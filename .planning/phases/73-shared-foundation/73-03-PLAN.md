### Task 3.5: Create usePermissions hook

<read_first>
- src/hooks/useAuth.ts (from Task 3.4 — usePermissions depends on useAuth)
- src/lib/permissions.ts (existing permission checking logic if any)
- .planning/phases/73-shared-foundation/73-SPEC.md (R10: usePermissions checks user roles)

<action>
Create `src/hooks/usePermissions.ts`:
1. Import `useAuth` from `./useAuth`
2. Hook signature: `usePermissions()` returns `{ can, cannot }`
3. Implementation:
   ```typescript
   export function usePermissions() {
     const { user } = useAuth();
     
     const can = useCallback((action: string, resource?: string): boolean => {
       if (!user) return false;
       
       // Check user role
       const role = user.role;
       
       // Permission matrix
       const permissions: Record<string, string[]> = {
         super_admin: ['*'], // All permissions
         coordinator_admin: ['read:requests', 'write:requests', 'assign:requests', 'read:users', 'read:audit'],
         reviewer: ['read:requests', 'review:requests', 'read:documents'],
         specialist: ['read:requests', 'write:requests', 'write:documents'],
         customer: ['read:requests', 'create:requests', 'read:own-requests'],
       };
       
       const userPermissions = permissions[role] || [];
       
       if (userPermissions.includes('*')) return true;
       
       const permissionKey = resource ? `${action}:${resource}` : action;
       return userPermissions.includes(permissionKey);
     }, [user]);
     
     const cannot = useCallback((action: string, resource?: string): boolean => {
       return !can(action, resource);
     }, [can]);
     
     return { can, cannot };
   }
   ```
4. Type exports: `UsePermissionsReturn` with `can` and `cannot` functions
5. Use `useCallback` for both functions to prevent re-renders

Add to `src/hooks/index.ts`:
- Export `usePermissions`
</action>
