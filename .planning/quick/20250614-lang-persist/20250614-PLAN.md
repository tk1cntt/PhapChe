# Quick Task: Persist Language Preference

## Problem
When user changes language in the header, navigating to another page or refreshing resets it back to Vietnamese (default locale).

## Solution
Save locale preference to browser localStorage when user changes language, and read it on app initialization.

## Tasks

### Task 1: Update LocaleProvider to read from localStorage
**files:** src/components/providers/LocaleProvider.tsx
**action:**
- Import `useEffect` and `useState` from React
- On mount, read `locale` from localStorage
- Set initial locale from localStorage if exists, otherwise use provided locale
- When locale changes, save to localStorage

**verify:** Component uses localStorage to persist and restore locale preference

**done:** LocaleProvider reads from localStorage on initialization

### Task 2: Update LanguageSwitcher to handle persistence
**files:** src/components/layout/LanguageSwitcher.tsx
**action:**
- Ensure LanguageSwitcher calls router.replace with new locale
- The persistence should happen via LocaleProvider reading updated value

**verify:** Changing language persists across page navigation

**done:** Language persists after navigation

## must_haves
- Locale preference saved to localStorage when changed
- Locale preference restored from localStorage on page load
- Works across all pages and navigation
- No visible delay when restoring preference
