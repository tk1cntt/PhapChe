## Fix 404 Error: Intake Submission Not Found

### Problem
When user creates a request draft and submits, the API `/api/intake/submit` returns 404 with error "Intake submission not found".

### Root Cause
- `create-draft/route.ts` creates only `LegalRequest` without `IntakeSubmission`
- `submitIntake()` in `intake-service.ts` looks for `IntakeSubmission` by `requestId`
- Result: Not found → 404 error

### Solution
Modified `create-draft/route.ts` to create `IntakeSubmission` alongside `LegalRequest`.

### Changes Made

1. **Fixed create-draft route** (`src/app/api/intake/create-draft/route.ts`):
   - Added `intakeSubmission.create` nested create in Prisma transaction
   - Now creates both `LegalRequest` and `IntakeSubmission` atomically

2. **Added e2e tests** (`e2e/intake-flow.spec.ts`):
   - Test: "creates request draft and submits without 404 error"
   - Test: "draft creation API returns requestId that can be used for submission"

3. **Added unit test** (`src/lib/intake/create-draft.test.ts`):
   - Test: "should create intakeSubmission when creating draft request"
   - Test: "should find intakeSubmission when submitting"
