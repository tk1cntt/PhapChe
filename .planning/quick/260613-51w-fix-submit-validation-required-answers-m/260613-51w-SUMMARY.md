## Summary

### Bug Fixed
**400 Error: "Required answers are missing"** - Occurred when submitting a request via CreateRequestForm.

### Root Cause
- `CreateRequestForm` doesn't have a step to collect intake answers (unlike legacy `/intake` flow)
- When submit is called, `IntakeSubmission.answers = {}`
- `validateAnswers()` in `intake-service.ts` fails validation for required fields

### Solution Applied
Modified `validateAnswers()` in `src/lib/intake/intake-service.ts` to skip validation when the matterType has no required questions:

```typescript
// Skip validation if matterType has no required questions (e.g., new wizard flow)
const requiredQuestions = matterType.questions.filter((question) => question.required);
if (requiredQuestions.length === 0) {
  return { ok: true, missingRequired: [] };
}
```

### Files Changed
1. `src/lib/intake/intake-service.ts` - Skip validation when no required questions
2. `e2e/intake-flow.spec.ts` - Updated tests to check for validation errors

### Verification
- Code review confirms validation is skipped for matterTypes without required questions
- E2E tests updated to verify no validation errors appear on submit
