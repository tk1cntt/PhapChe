## Critical Gaps

### G-1 (BLOCKER) — Reviewer service layer missing (REV-01..09)

- `src/lib/reviews/` does not exist; no review submission, approval, or rejection service.
- `/src/app/reviewer/requests/page.tsx` queries `prisma.documentVersion.findMany` with `legalRequest.assignedReviewerId` and `versionNumber`, both invalid against current schema — runtime Prisma error.
- Reviewer detail page is a stub: hardcoded `passedItemIds: string[] = []`, all TODOs, no real document load or checklist answer persistence.
- `DocumentVersion.status = 'final'` is unreachable from UI; the entire `approved → delivered → closed` flow cannot be triggered.
- Affects REQ-IDs: REV-01..09, DLV-01, DLV-02, DLV-04, DLV-05.

### G-2 (BLOCKER) — Specialist workbench version list broken (DOC-04 partial)

- `src/lib/documents/draft-service.ts:199-220` has a duplicate `document` key in the Prisma `select` for `listDocumentVersions`.
- Prisma throws on duplicate key; specialist workbench cannot render document version history.
- Affects REQ-IDs: DOC-04, DOC-06, REV-01.

### G-3 (HIGH) — VLT-05 folder/tag classification unimplemented

- `prisma/schema.prisma` has no Folder/Tag model, no service, no UI.
- 04-VERIFICATION recorded this as a failed must-have.
- Affects REQ-IDs: VLT-05.

### G-4 (MEDIUM) — Customer download UX (DLV-02 satisfied via redirect)

- Customer page renders bare `/api/vault/{id}/download`; the route now signs/redirects to a 15-minute signed URL.
- Works correctly, but every download requires an extra server round-trip and audit event.
- Recommend inlining signed URL in customer render.

### G-5 (MEDIUM) — Silent deliver/close form errors

- `markDeliveredAction`/`closeDeliveredAction` swallow errors and return `{ ok, message }` but the specialist page does not render the result.
- Transition itself works server-side; only UX feedback is missing.
